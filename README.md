# SeleniumHybridFramework

Repository: `git@github.com:Rakesh9254/SeleniumHybridFramework.git`

## Overview
A robust, scalable, and data-driven Selenium test automation framework built with Java and TestNG. This framework is designed to automate web applications with a focus on maintainability, modularity, and rich reporting. The framework currently contains end-to-end tests for [SauceDemo](https://www.saucedemo.com/).

## Tech Stack
* **Programming Language**: Java (JDK 11)
* **Automation Tool**: Selenium WebDriver 4.x
* **Test Runner**: TestNG 7.x
* **Build Tool**: Maven
* **Browser Management**: WebDriverManager
* **Logging**: Log4j2
* **Reporting**: ExtentReports 5.x

## Project Structure
* `src/main/java`: Contains Page Object classes, utilities, and core framework logic.
* `src/test/java`: Contains Test classes mapped to various modules (e.g., Login, Checkout).
* `testng.xml`: The main suite file to configure and execute test suites.
* `pom.xml`: Manages project dependencies and build plugins.
* `reports/`: Stores interactive HTML reports generated by ExtentReports.
* `logs/`: Stores application and test execution logs.
* `screenshots/`: Automatically captures and saves screenshots for failed test cases (`ScreenshotUtil`).

## How to Run Tests

### Using Maven
Navigate to the project root and run:
```bash
mvn clean test
```

### Using TestNG XML
Right-click on `testng.xml` in your IDE and select **Run as TestNG Suite**.

## Key Features
* **Cross-Browser Testing**: Configurable to run on Chrome, Firefox, and Safari via `@Parameters`.
* **Centralized Driver Configuration**: Implemented in `BaseTest.java` with automatic setup and teardown handling.
* **Failure Handling**: Listens for `ITestResult.FAILURE` and snaps visual evidence.
* **Readiness**: Utilizes Page Object Model (POM) design pattern for clean, reusable web elements separation.
